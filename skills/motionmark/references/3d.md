# 3D

Requires `perspective` in header. Use `deg` for all 3D angles.

## Header

```mmark
---
canvas: 640x360
bg: #0f172a
fps: 60
perspective: 620
vanish: 320 180
---
```

## Rotating Object

```mmark
@group world sort:depth | 0s - 6s
  x: 320
  y: 180
  rotateX: -18deg
  rotateY: 0deg -> 360deg over 6s linear

cuboid box w:120 h:80 d:90 fill:#38bdf8 parent:world | 0s - 6s
  frontFill: #3b82f6
  topFill: #93c5fd
```

## Views (flat surfaces in 3D)

```mmark
@view face parent:world | 0s - 6s
  at: 0 0 60
  normal: 0 0 1
  up: 0 -1 0

text label "Front" font:Roboto size:18 fill:#ffffff parent:face | 0s - 6s
  x: 0
  y: 0
```

`at`, `normal`, `up` are static (never animate them).

## Geometry

```mmark
line3d edge stroke:#ffffff strokeWidth:2 parent:world | 0s - 6s
  from: (0,0,-40)
  to: (80,20,40)

poly3d tri points:[(0,0,0), (60,0,0), (30,60,20)] fill:#ef4444 parent:world | 0s - 6s
```

## Solid Types

- `plane w h`
- `cuboid w h d`
- `sphere r segments rings`
- `cylinder r h segments`
- `cone r h segments`
- `pyramid r h sides`
- `prism r d sides`
- `torus major tube segments tubeSegments`

## 3D Props

Animatable: `z`, `rotateX`, `rotateY`, `rotateZ`, `depthBias`.
Static: `rotateOrder`, `sort`, `billboard`, `at`, `normal`, `up`, `from`, `to`, `points`.

## Rules

- Always use `deg` suffix: `rotateX: -18deg`, `rotateY: 0deg -> 360deg over 6s linear`.
- Put 3D objects in an `@group sort:depth`.
- `sort:depth` for solids, `sort:manual` for flat UI stacks.
- `billboard: screen` for labels that face camera.
