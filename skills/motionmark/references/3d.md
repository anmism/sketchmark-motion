# 2.5D and 3D Reference

Use this file for perspective scenes, 3D transforms, low-level 3D geometry, solid primitive shapes, sorting, and billboards. Read `3d-patterns.md` for longer example builds.

## Header Setup

```mmark
---
canvas: 640x360
bg: #0f172a
fps: 60
perspective: 620
vanish: 320 180
---
```

| Property | Description |
|---|---|
| `perspective` | Viewer distance from z=0. Lower values create stronger distortion. Typical: `500-900`. |
| `vanish` | Vanishing point, usually canvas center. |

## Core 3D Properties

| Property | Default | Animatable | Notes |
|---|---:|---:|---|
| `z` | `0` | yes | Positive is closer. |
| `rotateX` | `0` | yes | Pitch. |
| `rotateY` | `0` | yes | Yaw. |
| `rotateZ` | `0` | yes | Roll; combines with 2D `rotation`. |
| `scaleX`, `scaleY` | `1` | yes | Axis scale. |
| `rotateOrder` | `zyx` | no | 1-3 unique axes; missing axes are appended from `zyx`. |
| `origin` | `center` | no | Transform pivot. |
| `zIndex` | `0` | no | Layer ordering before depth ties. |
| `depthBias` | `0` | yes | Sort bias without moving geometry. |
| `sort` | `depth` | no | `depth`, `manual`, or `layer`. |
| `billboard` | `none` | no | `screen` or `y` keeps an element facing camera. |

Use `deg` for readable rotations:

```mmark
rotateY: 0deg -> 360deg over 6s linear
rotateX: -18deg
```

## Groups

Use `@group` to rotate or move a 3D object as one unit.

```mmark
@group world sort:depth | 0s - 8s
  x: 320
  y: 180
  rotateX: -18deg
  rotateY: 0deg -> 360deg over 8s linear
```

Child positions are relative to the group. A 3D parent rotates child positions and child geometry.

## Views

`@view` is a non-rendering oriented plane. Put normal 2D elements inside it to build faces, walls, floors, and UI planes.

```mmark
@view front parent:cube | 0s - 8s
  at: 0 0 45
  normal: 0 0 1

rect face w:90 h:90 fill:#3b82f6 parent:front | 0s - 8s
```

| Property | Description |
|---|---|
| `at` | Plane center in parent 3D space, as `x y z`. |
| `normal` | Plane normal vector; default `0 0 1`. |
| `up` | Plane up vector; default `0 -1 0`. |

`at`, `normal`, and `up` must be static.

## Low-Level 3D Primitives

```mmark
line3d edge stroke:#ffffff strokeWidth:2
  from: (0, 0, -40)
  to: (80, 20, 40)

poly3d face fill:#ef4444 stroke:#ffffff22 strokeWidth:1
  points: [(0,0,0), (60,0,0), (30,60,20)]

path3d trail d:"M0,0,0 C40,-40,80 100,40,-80 140,0,0" fill:none stroke:#38bdf8 strokeWidth:5
  strokeCap: round
```

| Primitive | Use for | Required static geometry |
|---|---|---|
| `line3d` | Axes, bonds, edges, connectors | `from`, `to` |
| `poly3d` | Custom planar faces | `points` |
| `path3d` | Curves, trails, ribbons | `points` or 3D `d` |

`path3d d` supports 3D `M`, `L`, `Q`, `C`, and `Z` commands with x/y/z coordinates.

## Solid Primitives

Prefer solid primitives when they fit. They compile into generated `poly3d` faces with stable sorting.

```mmark
plane floor w:500 h:320 fill:#111827
  rotateX: 90deg

cuboid box w:120 h:80 d:90 fill:#38bdf8
  frontFill: #3b82f6
  topFill: #93c5fd

sphere planet r:70 segments:16 rings:9 fill:#34d399
  middleFill: #34d399
  bottomFill: #065f46

cylinder can r:42 h:90 segments:16 fill:#38bdf8
  side0Fill: #7dd3fc

cone marker r:46 h:100 segments:16 fill:#f97316
pyramid roof r:54 h:90 sides:4 fill:#ef4444
prism wedge r:52 d:86 sides:3 fill:#14b8a6
torus ring major:52 tube:12 segments:18 tubeSegments:6 fill:#fb923c
```

| Primitive | Size props | Face overrides |
|---|---|---|
| `plane` | `w`, `h` | `faceFill`, `faceStroke`, `faceStrokeWidth`, `faceOpacity` |
| `cuboid` | `w`, `h`, `d` | `front*`, `back*`, `left*`, `right*`, `top*`, `bottom*` |
| `sphere` | `r`, `segments`, `rings` | `top*`, `upper*`, `middle*`, `lower*`, `bottom*`, `faceN*` |
| `cylinder` | `r`, `h`, `segments` | `top*`, `bottom*`, `sideN*` |
| `cone` | `r`, `h`, `segments` | `base*`, `sideN*` |
| `pyramid` | `r`, `h`, `sides` | `base*`, `sideN*` |
| `prism` | `r`, `d`, `sides` | `front*`, `back*`, `sideN*` |
| `torus` | `major`, `tube`, `segments`, `tubeSegments` | `faceN*` |

Material override suffixes: `Fill`, `Stroke`, `StrokeWidth`, `Opacity`.

## Sorting

Use `sort` on groups when depth behavior matters.

```mmark
@group stack sort:manual | 0s - 6s
  x: 320
  y: 180
```

| Mode | Behavior |
|---|---|
| `depth` | Sort by `zIndex`, then average transformed z-depth. |
| `manual` | Keep declaration order inside the group. |
| `layer` | Use `zIndex`, then declaration order. |

Use `depthBias` for halos, labels, and outlines that must sit above a surface without changing the geometry.

## Billboards

```mmark
text label "Node" font:Roboto size:18 fill:#e2e8f0 parent:world | 0s - 8s
  x: 0
  y: -120
  z: 60
  billboard: screen
```

`billboard: screen` keeps the element facing the screen while inheriting position. `billboard: y` is accepted by validation; use for y-axis-facing labels.

## Guidance

- Start with `perspective: 600` and vanish at canvas center.
- Put whole 3D objects in an `@group`; animate the group rotation.
- Use built-in solids before hand-building face stacks.
- Use `@view` for panels that need child text or normal 2D shapes.
- Use `line3d`, `poly3d`, and `path3d` for technical diagrams and custom geometry.
- Lower `segments`/`rings` for low-poly style and performance.
- Use `billboard: screen` for readable labels inside rotating scenes.
- Use `sort: manual` for coplanar UI-like stacks that flicker under depth sorting.
